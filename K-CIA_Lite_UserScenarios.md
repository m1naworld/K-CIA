# K-CIA Lite — 공공데이터 기반 유저 시나리오/데이터 설계 (Notion 공유용)

## 1) PRD 요약 (12줄 이내)

- **K-CIA Lite**는 성수동을 대상으로 **공공데이터 기반 정량 팩트**를 3D Hex 지도+대화로 제공하는 하이퍼로컬 상권 분석 플랫폼입니다.
- 사용자는 복잡한 엑셀/BI 없이 **지도에서 “핫스팟을 발견(탐색)”**하고, 클릭/질의로 **“왜/어떻게”를 의사결정 수준으로** 얻습니다.
- 핵심 UI는 **3D Hex 지도(높이=유동/매출 등)** + **사이드바 요약지표** + **LangGraph 멀티에이전트 챗봇**입니다.
- 공공데이터는 **분기·월 단위 후행성**이 있어, UX에서 **“데이터 기준 시점” 라벨**과 **보조지표(점포변화/집객시설/실시간 혼잡 등)**로 보정합니다.
- 결과는 “추천+근거+리스크+액션”으로 구조화해 **저장/공유/보고서**로 전환합니다.
- (옵션) 유튜브/소셜은 분리 모듈로 두고, **없어도 추천이 성립**하도록 설계합니다.
- KPI는 **자연어→SQL 성공률, 질의 반복, 분석 저장/공유 전환율** 중심입니다.

## (옵션) 유튜브/소셜 모듈 — **없어도 성립**, 있으면 “후행성 보정 + 설명력 강화”

> 공공데이터(분기/월)의 후행성을 보정하고, “왜 이런 변화가 생겼는지”를 정성 근거로 보강하는 모듈입니다.
> 
> 
> **기본 정책:** 유튜브 모듈이 꺼져 있어도 모든 시나리오는 공공데이터만으로 완료됩니다.
> 

### 사용자가 화면에서 보는 것

- 지도 우측 토글: `정량(유동/매출)` ↔︎ `정성(유튜브 반응)`
- 정성 레이어(색상): `긍·부정 감성`, `언급량(버즈)`, `주제(키워드)`
- 사이드바 카드: **최근 7일/30일 트렌드**, “주요 키워드 Top10”, “대표 코멘트(요약)”
- 챗봇 응답: **정량 근거 + 정성 근거**를 분리 표기(근거/주의/대안)

### 데이터가 없을 때(=유튜브 OFF) 답변 전략

- “후행성 경고(as-of 배지)” + 공공 보조지표(D11 실시간 혼잡/교통 등)로 보정
- “왜(원인)” 질문에는 **단정 금지**: 가능한 설명 + 확인 체크리스트(현장/리뷰/검색) 제공

### 데이터가 있을 때(=유튜브 ON) 최소 산출물

- `sentiment_score ∈ [-1,1]`, `buzz_volume`, `topic_keywords`, `evidence_snippets`(요약)
- 지도 레이어는 **H3에 직접 고정 매핑이 어려우면** “성수동 범위(지역 스코프)” 수준으로 먼저 제공(점진적 고도화)

## 4) 공공데이터 기반 유저 시나리오 8개 (각 6단계 + 데이터 사용 설계)

> 공통 UX 규칙
> 
> 
> - 모든 카드/표/차트에 `데이터 기준시점(as-of)`와 `신선도 배지(분기/월/실시간)` 표시
> 
> - “최근/요즘/오늘” 질의는 자동으로 **D11(실시간)**를 함께 조회해 **후행성 보정 카드**를 동봉
> 
> - **Q1(확정)** 성수동 범위는 **행정동 + 상권코드 둘 다 지원**
> 
> - UI: `영역 기준(행정동/상권)` 토글
> 
> - 기본 프리셋: **행정동(성수1가1/성수1가2/성수2가1/성수2가3)** ∪ **골목상권(연무장길/성수역/뚝섬역 인근 상권코드)**
> - **Q2(확정)** Hex 기본 해상도는 **H3 res=9**(성능/해석성 균형)
> 
> - 성능 문제 시 res=8(가벼움) / 정밀분석 시 res=10(촘촘) 실험 플래그 제공
>

### 시나리오 1 (P1, 32세) 성수동 디저트 카페 **입지 Top3 추천**

### 1) 탐색

- **(유튜브 옵션)**: 지도 색상 토글 `유튜브 반응(버즈/감성)`을 켜면, 후보별 “최근 7일 키워드/대표 코멘트 요약” 카드가 추가됩니다.
- **UI**: 3D Hex 지도(높이=유동인구, 색=매출 증감) + 좌측 필터(업종=디저트/카페, 분기=최근4개)
- **데이터셋 후보**: D5(유동), D1(매출)
- **핵심 필드(예시)**
    - D5: `기준_년분기_코드`, `상권_코드`, `시간대`, `요일`, `유동인구_수`
    - D1: `기준_년분기_코드`, `상권_코드`, `서비스_업종_코드`, `추정_매출_금액`, `추정_매출_건수`
- **조인키/시간키**: `상권_코드` + `기준_년분기_코드` (업종은 `서비스_업종_코드`)
- **집계 로직**: 상권→Hex(H3) 매핑 후 `유동/매출` 표준화(z-score) + 가중합 → TopN Hex
- **갱신/후행 UX**: 분기 후행(약 1~3개월) → 카드에 “최근 분기 기준(후행)” 배지 + D11(실시간) 유도 버튼

### 2) 선택

- **UI**: Hex 클릭 → 사이드바 “요약 카드 6개(유동, 매출, 경쟁, 성장, 리스크, 추천)” + 미니차트(QoQ)
- **데이터셋 후보**: D1 + D2 + D5
- **핵심 필드**: D2 `점포_수`, `개업_점포_수`, `폐업_점포_수`
- **조인키/시간키**: (상권 기준) D2는 행정동 기반이므로 `상권영역(D3) ↔︎ 행정동(D9)` 공간조인으로 근사 매핑
- **집계 로직**: 동일 업종 점포수(경쟁) + 매출QoQ + 유동QoQ → “기회/리스크” 분해
- **갱신/후행 UX**: “경쟁(점포)”도 분기 집계 → “최근 분기” 라벨 고정

### 3) 질의

- **UI**: 챗봇 “여기 디저트 카페 차려도 될까? (초기 30평, 주말 집중)”
- **데이터셋 후보**: D1 + D2 + D5 (필요 시 D4)
- **핵심 필드**: `면적(가정)`, `객단가(가정)`, `주말비중(계산)`
- **조인키/시간키**: `상권_코드`, 최근 4개 분기
- **집계 로직**: 유동(주말/시간대)과 매출/경쟁을 결합해 “적합도 점수” 산출
- **갱신/후행 UX**: “운영 가정값(면적/객단가)” 입력 시 **가정 배지** 표시

### 4) 분석

- **UI**: 근거 카드(Top3) + 리스크 카드(Top2) + 대안(Top2 Hex)
- **집계 로직**: `score = a*z(유동) + b*z(매출) - c*z(경쟁) + d*z(성장)`*(기본 가중치 제공, 사용자 슬라이더로 조정)*

### 5) 결과

- **UI**: Top3 후보 + “왜 이곳인가” 3줄 + 체크리스트(실사 포인트 6개)
- **UX 보정**: 분기 후행 경고 + 실시간 혼잡 확인 버튼(D11)

### 6) 액션(저장/공유/보고서)

- **UI**: “입지 후보 저장” + “PDF 공유” + “골든 스냅샷 저장”
- **저장 모델**: `analysis_run`에 질문/필터/SQL/결과/가정/데이터 기준시점 저장

### 시나리오 2 (P1, 32세) **피크타임 기반 운영전략** (영업시간/인력/메뉴)

### 1) 탐색

- **UI**: 지도 토글 “시간대 유동 히트맵(평일/주말)”
- **데이터셋 후보**: D5
- **핵심 필드**: `시간대`, `요일`, `유동인구_수`, `연령대`, `성별`
- **조인키/시간키**: `상권_코드` + `기준_년분기_코드`
- **집계 로직**: 시간대/요일별 유동 피크 추출(Top3 시간대)

### 2) 선택

- **UI**: 선택 구역 사이드바에 “피크타임 3개/오프피크 2개” 자동 표시
- **데이터셋 후보**: D5 + (가능 시) D1
- **집계 로직**: 피크 시간대 유동비중 vs (업종) 매출비중 비교

### 3) 질의

- **UI**: “오후 3~5시 인력 줄여도 돼?”
- **데이터셋 후보**: D5 + D1 (보정: D11)
- **집계 로직**: 오프피크의 유동/매출 기여도 → “권장 인력 변화(상대)” 계산
- **후행 UX**: D11 실시간이 분기 평균과 크게 다르면 “변동성 경고” 배지

### 4) 분석

- **UI**: 시간대 차트(유동/매출) + “권장 스케줄” 카드
- **집계 로직**: `labor_reco_score = 매출기여도 - (유동대비 과잉 추정)` *(객단가/회전율은 입력 가정값)*

### 5) 결과

- **UI**: 추천 2개(인력/프로모션) + 리스크 2개(이벤트/날씨/경쟁)
- **UX 보정**: “가정 기반” 표기(객단가/회전율/좌석 수)

### 6) 액션

- **UI**: “운영 플랜 저장/공유/PDF”
- **저장 모델**: 운영 파라미터(`assumptions_json`) 포함 스냅샷 저장

### 시나리오 3 (P2, 29세) **팝업 스토어 위치·기간·시간대 결정**

### 1) 탐색

- **(유튜브 옵션)**: “팝업/브랜드/핫플” 키워드 버즈(최근 7일)와 감성 요약을 후보 비교 카드에 동봉해 내부 설득용 근거로 사용합니다.
- **UI**: 팝업 모드(필터: 타겟 연령/성별, 주말 강조) + 3D Hex(높이=유동)
- **데이터셋 후보**: D5
- **핵심 필드**: `연령대별_유동`, `성별_유동`, `요일`, `시간대`, `유동인구_수`
- **조인키/시간키**: `상권_코드` + `기준_년분기_코드`
- **집계 로직**: 타겟층 유동비중(예: 20~34) 가중치 적용 → 후보 Hex 랭킹

### 2) 선택

- **UI**: 후보 Hex 클릭 → “타겟 유입 시간대 Top3” + “근거 표(유동/성별/연령)”
- **데이터셋 후보**: D5 + D8(집객시설)

### 시나리오 8 (P4, 47세) **신규 상가 투자/임대 검토(A/B)** (수요+리스크+가격 근거)

- **조인키/시간키**:
    - 수요/경쟁: `상권_코드 + 기준_년분기_코드`
    - 가격: `법정동코드(앞5자리) + 계약년월(YYYYMM)`
    - (필요 시) 법정동↔︎행정동은 공간조인(D9) 또는 코드 매핑 테이블로 근사

### 3) 질의

    - **수요 점수**: 유동/매출 z-score
    - **경쟁/이탈**: 점포/폐업 변화율
    - **가격 레퍼런스**: D14 최근 6~12개월 거래 단가(원/㎡) 분포
    - **거시 리스크**: D15 공실률/수익률 배지

### 4) 분석

- **UI**: A/B 점수표 + “민감도(임대료 10%↓, 공실 1개월↑)” 시뮬레이션 카드
- **후행 UX**: 미시 임대료/공실은 공공데이터로 제한 → **사용자 입력(가정)** + **거시 배지(D15)**로 보정

### 5) 결과

- **UI**: 결론(추천 1안) + 리스크 2개 + 대안 1개(인근 후보)
- **근거**: KPI 3개 + 가격 레퍼런스 1개 + 기준시점 표시

### 6) 액션

- **UI**: “투자 검토 메모(1페이지)” 저장/공유 + 골든 스냅샷
- **저장 모델**: `investment_memo`(가정/근거/결론) + `analysis_run` 참조
